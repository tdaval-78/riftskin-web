macOS bloque RIFTSKIN : comment autoriser l'application

Sur Mac, il est normal qu'un installateur non distribué via l'App Store ou signé selon un flux grand public déclenche des écrans de sécurité. Ce n'est pas automatiquement un bug RIFTSKIN. Le plus souvent, il faut simplement autoriser le paquet dans les réglages macOS.

Les messages les plus fréquents

  • Apple cannot verify that it is free from malware.
  • Open Anyway dans Privacy & Security.
  • Demande de mot de passe administrateur ou Touch ID.
  • macOS refuse d'ouvrir le paquet au premier essai.

La bonne lecture du problème

Dans la majorité des cas, cela signifie simplement que macOS vous demande une validation manuelle. Tant que vous utilisez l'installeur officiel RIFTSKIN, le bon réflexe est de suivre le flow d'autorisation, pas de penser immédiatement que l'application est cassée.

Étapes pour autoriser RIFTSKIN

01

Téléchargez l'installeur officiel

Utilisez toujours la page officielle de téléchargement pour récupérer le paquet macOS le plus récent.

02

Essayez de l'ouvrir une première fois

L'échec initial est souvent normal. Il sert surtout à faire apparaître l'option d'autorisation dans les réglages système.

03

Allez dans Confidentialité et sécurité

Ouvrez Réglages système, puis Confidentialité et sécurité. Cherchez la ligne liée à RIFTSKIN ou à l'installeur bloqué.

04

Cliquez sur Ouvrir quand même

Validez l'autorisation, puis confirmez avec le mot de passe administrateur ou Touch ID si macOS le demande.

Quand il faut réessayer l'installeur

Après autorisation, il faut généralement relancer le paquet une nouvelle fois. C'est cette deuxième ouverture qui permet de poursuivre normalement l'installation jusqu'à la copie dans Applications.

Quand contacter le support

  • Si aucune option Ouvrir quand même n'apparaît après plusieurs essais.
  • Si l'installeur échoue encore après autorisation.
  • Si l'application s'ouvre puis plante immédiatement.
  • Si le problème survient après une mise à jour de macOS ou du jeu et que le comportement n'est plus le même qu'avant.
